Media Query en CSS

Diseño responsive avanzado y estrategias de maquetación profesional

¿Qué es un Media Query?

Un Media Query es una función de CSS que permite adaptar el diseño de una página web según el tamaño de la pantalla, la resolución o la orientación del dispositivo.

Gracias a esta herramienta, las páginas pueden verse correctamente en celulares, tablets y computadores desde un mismo código base.

Anatomía de una Media Query

Una Media Query se compone de un tipo de medio (opcional), una o más expresiones de características (como el ancho), y los estilos CSS que se aplicarán si la condición es verdadera. 

                                                                                                                                            /* Estructura: @media [tipo de medio] and (característica) */
                                                                                                                                            @media screen and (max-width: 768px) {
                                                                                                                                            body {
                                                                                                                                                background: lightblue;
                                                                                                                                                    }
                                                                                                                                                    }

Tipos de Medios (Media Types)

  • all: Se usa para todos los dispositivos (valor por defecto).
  • screen: Destinado a pantallas de computadores, tablets, celulares, etc.
  • print: Destinado a material impreso y páginas vistas en pantalla en modo de vista previa de impresión.

Operadores Lógicos

Puedes crear reglas más específicas combinando condiciones con operadores lógicos:

  • and: Combina múltiples condiciones. Todas deben cumplirse. (Ej: Pantallas que sean de tipo screen y tengan un ancho mínimo de 600px).
  • not: Invierte el resultado de la Media Query.
  • , (coma): Funciona como un operador OR. Aplica los estilos si al menos una de las condiciones se cumple.
                                                                                                                                                                                                                            /* Ejemplo con rango: Solo pantallas entre 600px y 1000px */
                                                                                                                                                                                                                            @media screen and (min-width: 600px) and (max-width: 1000px) {
                                                                                                                                                                                                                            .columna {
                                                                                                                                                                                                                                width: 50%;
                                                                                                                                                                                                                                    }
                                                                                                                                                                                                                                    }

Estrategias de Diseño: Mobile-First vs. Desktop-First

A la hora de maquetar existen dos filosofías principales. La industria actual prefiere mayoritariamente Mobile-First debido al alto tráfico desde smartphones.

Característica Mobile-First (Recomendado) Desktop-First (Tradicional)
Punto de partida Diseño para pantallas pequeñas. Diseño para monitores grandes.
Propiedad CSS Usa principalmente min-width. Usa principalmente max-width.
Rendimiento móvil Excelente (carga menos código base). Más lento (descarga estilos pesados innecesarios).

Breakpoints Estándar en la Industria

Aunque los puntos de ruptura cambian según el diseño, estos son los estándares que utilizan frameworks como Bootstrap:

  • Dispositivos extra pequeños (Móviles): Menor a 576px.
  • Dispositivos pequeños (Móviles grandes): 576px o más.
  • Dispositivos medianos (Tablets): 768px o más.
  • Dispositivos grandes (Laptops): 992px o más.
  • Dispositivos extra grandes (Monitores): 1200px o más.

Ventajas del Responsive Design

  • Mejora la experiencia del usuario (UX): Evita el molesto zoom horizontal en pantallas móviles.
  • Beneficios de SEO: Google indexa y posiciona prioritariamente las páginas optimizadas para móviles.
  • Código unificado: Un solo archivo HTML y CSS sirve para cualquier pantalla existente.
